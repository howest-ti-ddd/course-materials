DDD Strategic Patterns

A Quick Reference Guide

Core Concepts

Domain

The business problem space. What the software is for.

Bounded Context

A boundary where a specific model is consistent. Protects the model's integrity.

Ubiquitous Language

A shared, rigorous language between developers and experts, used in code.

Subdomain Patterns

Core

Core Domain

The business's competitive advantage. Assign your best team.

Sup

Supporting

Necessary, but not a differentiator. Build simply or outsource.

Gen

Generic

A "solved problem" not specific to the business. Always buy, never build.

Flow / Direction

U

Upstream (U)

The context that influences or provides a service to another.

D

Downstream (D)

The context that is influenced by or consumes a service.

Context Map Relationships

P

Partnership

Two teams succeed or fail together. High coordination needed.

SK

Shared Kernel

Contexts share a small, co-owned model subset. Keep it tiny!

C/S

Customer-Supplier

Upstream (Supplier) prioritizes needs of Downstream (Customer).

CF

Conformist

Downstream blindly adopts the Upstream model. No translation.

ACL

Anticorruption Layer

Downstream builds a translation layer to protect its model.

OHS

Open Host Service

A context offers a public API for many consumers.

PL

Published Language

A documented, shared data format for an OHS.

SW

Separate Ways

Contexts are not integrated as there is little value in doing so.
