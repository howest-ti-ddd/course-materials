Learning Objectives

Lesson: Domains, Bounded Contexts & Context Mapping

While reviewing the course materials, focus on mastering these key learning objectives.

Bounded Context Fundamentals

Define and identify Bounded Contexts

  • • Explain the primary purpose of Bounded Contexts as linguistic and conceptual boundaries
  • • Detect implicit Bounded Context boundaries through terminology conflicts

Distinguish between Problem and Solution space

  • • Differentiate between Subdomains (problem space) and Bounded Contexts (solution space)
  • • Explain the optimal relationship between Core Subdomains and Bounded Contexts

Context Mapping Patterns

Apply defensive integration patterns

  • • Evaluate when Conformist patterns are appropriate despite model limitations

Design service-oriented integration patterns

  • • Analyze when Separate Ways is the strategically sound decision

Strategic Domain-Driven Design

Create and analyze Context Maps

  • • Visualize socio-technical relationships between teams and bounded contexts
  • • Identify improvement opportunities through context map analysis

Classify and prioritize Subdomains

  • • Distinguish between Core, Supporting, and Generic Subdomains
  • • Allocate resources appropriately based on subdomain classification

Preserve model integrity

  • • Identify and resolve linguistic ambiguities and False Cognates

Organizational Alignment

Apply Conway's Law strategically

  • • Explain how communication structures constrain system designs
  • • Design team structures that facilitate desired architectural outcomes
