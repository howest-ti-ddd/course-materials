Microkernel: Trade-offs

When to Use

  • Products needing third-party extensibility (VS Code, Jenkins, Eclipse).
  • Systems with a generic core and highly variable rules (tax software, insurance claims).
  • Applications where features need to be enabled/disabled at runtime.

When Not to Use

  • When the "core" functionality is not stable and changes often.
  • When plug-ins need to communicate directly with each other.
  • Simple apps that don't need high extensibility.
