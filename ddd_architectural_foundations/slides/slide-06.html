DDD Recap: Ubiquitous Language

What is it?

A common vocabulary shared between developers and domain experts. Everyone speaks the same language about the business.

Key Rule:

If domain experts don't use a term, neither should your code.

❌ Bad Example

class DataProcessor {
  processUserData(input)
}

✅ Good Example

class OrderProcessor {
  processCustomerOrder(order)
}

In Howestprime project: Use terms like "Visitor", "Ticket", "Movie" - not "User", "Item", "Container".
