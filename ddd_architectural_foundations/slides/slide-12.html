Software Design vs Software Architecture

Software Design

Focus: How individual components work internally

Scope: Classes, methods, algorithms, data structures

Questions: How should this class be implemented? What design patterns should I use?

Examples: "Use the Strategy pattern for payment processing" or "Make this class immutable"

Focus: How major components interact with each other

Scope: System structure, component boundaries, deployment

Questions: Should this be a monolith or microservices? How do we handle scalability?

Examples: "Split the system into Order and Payment services" or "Use event-driven architecture"

Think of it like building a house: Architecture is the floor plan, Design is how you wire the electricity.
