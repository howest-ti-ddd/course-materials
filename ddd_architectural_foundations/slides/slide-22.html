Key Fallacies: Reliability & Latency

Fallacy: The Network Is Reliable

In reality, networks fail. Packets are lost. Connections drop. A service might be perfectly healthy but unreachable.

Impact: We must design for failure using patterns like retries, timeouts, and circuit breakers.

Fallacy: Latency Is Zero

A local function call takes nanoseconds. A network call takes milliseconds, which is thousands of times slower. It's never zero.

Impact: Every remote call adds delay. Architectures with "chatty" services become slow. We must minimize cross-service calls.
